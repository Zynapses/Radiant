Part 2: Method Details with Parameters

GENERATE_RESPONSE

Common: Generate | Scientific: Basic Generation

  • Inputs: prompt (string), context (string, optional)
  • Output: response (string), tokens (number)
  • Parameters:
    • temperature: 0-2 (default: 0.7)
    • max_tokens: integer (default: 4096)
  • Model Role: generator
  • Recommended Models: Claude 3.5 Sonnet, GPT-4o, DeepSeek Chat

GENERATE_WITH_COT

Common: Think Step-by-Step | Scientific: Chain-of-Thought Generation

  • Inputs: prompt (string)
  • Output: response (string), reasoning (string)
  • Parameters:
    • temperature: 0-2 (default: 0.3)
    • max_tokens: integer (default: 8192)
    • thinking_budget: integer (default: 2000)
  • Model Role: generator
  • Recommended Models: o1, DeepSeek Reasoner, Claude 3.5 Sonnet

CRITIQUE_RESPONSE

Common: Critique | Scientific: Critical Evaluation

  • Inputs: original_prompt (string), response (string)
  • Output: issues (array), suggestions (array), severity_scores (object)
  • Parameters:
    • focus_areas: array (default: ["accuracy", "completeness", "clarity", "logic"])
    • severity_threshold: "low" | "medium" | "high" (default: "medium")
  • Model Role: critic
  • Recommended Models: o1, Claude 3.5 Sonnet

JUDGE_RESPONSES

Common: Judge | Scientific: Comparative Judgment

  • Inputs: original_prompt (string), responses (array of strings)
  • Output: best_index (number), score (0-1), reasoning (string)
  • Parameters:
    • evaluation_mode: "pointwise" | "pairwise" | "listwise" (default: "pairwise")
    • criteria: array (default: ["accuracy", "helpfulness", "clarity", "completeness"])
  • Model Role: judge
  • Recommended Models: o1, Claude 3.5 Sonnet

VERIFY_FACTS

Common: Fact Check | Scientific: Factual Verification

  • Inputs: response (string)
  • Output: claims (array), verdicts (array), confidence (number)
  • Parameters:
    • extraction_method: "explicit" | "implicit" (default: "explicit")
    • verification_depth: "quick" | "thorough" | "exhaustive" (default: "thorough")
  • Model Role: verifier
  • Recommended Models: o1, Claude 3.5 Sonnet

SYNTHESIZE_RESPONSES

Common: Synthesize | Scientific: Multi-Response Synthesis

  • Inputs: original_prompt (string), responses (array with model attribution)
  • Output: synthesized_response (string), sources (array)
  • Parameters:
    • combination_strategy: "best_parts" | "merge_all" | "hierarchical" (default: "best_parts")
    • conflict_resolution: "majority" | "most_confident" | "all" (default: "majority")
  • Model Role: synthesizer
  • Recommended Models: Claude 3.5 Sonnet, GPT-4o

BUILD_CONSENSUS

Common: Consensus | Scientific: Consensus Aggregation

  • Inputs: responses (array of strings)
  • Output: consensus_points (array), disputed_points (array), unique_insights (array)
  • Parameters:
    • consensus_threshold: 0-1 (default: 0.7)
    • include_disputed: boolean (default: true)
  • Model Role: synthesizer
  • Recommended Models: Claude 3.5 Sonnet, GPT-4o

DECOMPOSE_PROBLEM

Common: Decompose | Scientific: Problem Decomposition

  • Inputs: prompt (string)
  • Output: subproblems (array), dependencies (graph), complexity_estimates (object)
  • Parameters:
    • max_subproblems: integer (default: 5)
    • decomposition_strategy: "functional" | "temporal" | "hierarchical" (default: "functional")
  • Model Role: reasoner
  • Recommended Models: o1, Claude 3.5 Sonnet

SELF_REFLECT

Common: Reflect | Scientific: Self-Reflection

  • Inputs: original_prompt (string), response (string)
  • Output: strengths (array), weaknesses (array), improvements (array), revised_response (string)
  • Parameters:
    • reflection_depth: "quick" | "thorough" | "exhaustive" (default: "thorough")
    • aspects: array (default: ["accuracy", "completeness", "clarity"])
  • Model Role: critic
  • Recommended Models: Claude 3.5 Sonnet, o1

Part 5: Parallel Execution Configuration

parallelExecution: {
  enabled: boolean,              // Enable parallel execution
  mode: 'all' | 'race' | 'quorum', // Wait strategy
  models: string[],              // Models to use (or let AGI select)
  quorumThreshold?: number,      // For quorum mode (0.5 = majority)
  synthesizeResults?: boolean,   // Combine results
  synthesisStrategy?: 'best_of' | 'merge' | 'vote' | 'weighted',
  timeoutMs?: number,           // Timeout per call
  failureStrategy?: 'fail_fast' | 'continue' | 'fallback',
  
  // AGI Dynamic Selection
  agiModelSelection?: boolean,   // Let AGI pick models
  minModels?: number,           // Min models to select (default: 2)
  maxModels?: number,           // Max models to select (default: 5)
  
  // Output Streaming
  outputMode: 'single' | 'all' | 'top_n' | 'threshold',
  outputTopN?: number,          // For top_n mode
  outputThreshold?: number,     // Min confidence for threshold mode
}

Part 8: Usage Examples

Simple Workflow Execution

const result = await orchestrationPatternsService.executeWorkflow({
  tenantId: 'tenant-123',
  userId: 'user-456',
  workflowCode: 'CoT',  // Chain-of-Thought
  prompt: 'Solve this math problem...',
  configOverrides: {
    temperature: 0.3,
    max_iterations: 2
  }
});

AGI Auto-Selection

const plan = await agiBrainPlannerService.generatePlan({
  tenantId: 'tenant-123',
  userId: 'user-456',
  prompt: 'Complex research question...',
  allowAgiWorkflowSelection: true,  // AGI picks best workflow
  enableConsciousness: true,
  enableEthicsCheck: true,
});
// plan.selectedWorkflow - AGI's choice
// plan.workflowSteps - Configured steps

Parallel Multi-Model with AGI Selection

// In workflow step configuration:
{
  stepOrder: 2,
  methodCode: 'GENERATE_RESPONSE',
  parallelExecution: {
    enabled: true,
    agiModelSelection: true,  // AGI picks models
    minModels: 2,
    maxModels: 4,
    synthesizeResults: true,
    synthesisStrategy: 'best_of',
    outputMode: 'single'  // Synthesized into one response
  }
}
